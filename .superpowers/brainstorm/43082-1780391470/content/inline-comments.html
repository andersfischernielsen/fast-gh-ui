Inline code comments in diffs

GitHub supports commenting on specific lines in a diff. How should we handle this?

A

Read-only viewing

Show existing inline comments inline in the diff. No writing new comments from this app — open PR on GitHub to comment.

B

Full read + write

View and create inline comments, start review threads, reply to existing ones. Full PR review capability.

C

Start read-only, add write later

Build read-only first. Writing inline comments is phase 2. Unblocks everything else sooner.
