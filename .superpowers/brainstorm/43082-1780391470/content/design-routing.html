Routing & State Management

Routes

/login                — TokenGate (enter/save GitHub token)
/                     — Notifications list (default view)
/@[owner]/[repo]/pull/[number]  — PR detail view
/notifications         — (alias to /)
/settings              — token management, preferences

State Management

Svelte 5 runes in a lightweight store pattern — no external state library needed:

src/lib/
├── stores/
│   ├── token.svelte.ts      — $state for token, persisted to localStorage
│   ├── notifications.svelte.ts — $state for notification list + filters
│   └── current-pr.svelte.ts — $state for active PR data
├── github/                  — (data layer, covered above)
└── components/              — (UI components)

Each store is a .svelte.ts module exporting reactive state via runes. Components import and use them directly — no providers, no context tree.

Token Flow

  1. User enters token on /login
  2. Stored in localStorage, loaded into Svelte $state on app init
  3. +layout.svelte checks if token exists → redirect to /login if missing
  4. Octokit client created from token in browser — never sent to server
  5. Logout clears localStorage + redirects to /login
