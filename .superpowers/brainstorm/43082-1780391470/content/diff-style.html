PR reading experience: diff approach

How should we render PR diffs? This is the biggest performance + UX decision.

A

Split diff (GitHub-style)

Two columns: old vs new. Familiar to GitHub users. Richer visual diff but more horizontal space needed.

B

Unified diff (git-style)

Single column with +/- markers. Compact, fast to scan, great for keyboard users. More lines visible at once.

C

Both, toggleable

Default to unified (performance), toggle to split on demand. Best of both worlds.
