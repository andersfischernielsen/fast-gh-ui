Notifications viewing pattern

How should notifications be organized and navigated?

A

Simple chronological list

Flat list, newest first. Filter by repo/type. Click to open the linked PR/issue. Like GitHub notifications page but faster.

B

Grouped by repository

Notifications grouped by repo, then by thread. Better for triaging per-repo. Collapse/expand groups.

C

Inbox-zero style

Thread-based grouping (like email). Mark done/snooze/archive. Focus on processing, not just reading.
