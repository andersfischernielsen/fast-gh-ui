Data & caching strategy

Performance is a priority. How should we handle GitHub API data?

A

Live-only (minimal caching)

Fetch fresh data every time. Use GitHub's ETag/conditional requests to avoid re-downloading. Simplest, but slower on repeated views.

B

IndexedDB cache layer

Cache API responses in IndexedDB. Show cached data instantly, refresh in background. Much faster for navigation, more complex.

C

Service Worker + Cache API

Service Worker intercepts Octokit calls, caches responses. Works offline, instant repeat views. Most complex but best perf.
